# Work Log

This folder tracks project progress so work can continue from any device.

## Files

- `CURRENT_STATUS.md` - canonical current state, last completed ticket, next step.
- `YYYY-MM-DD-session.md` - chronological session notes for that day.

## Update Rules

1. Update `CURRENT_STATUS.md` after each completed ticket or important decision.
2. Append a new entry to today's session file after each meaningful work block.
3. Include commit hash when changes are committed.
4. Keep entries short and actionable.

## Resume Workflow

1. Open `CURRENT_STATUS.md`.
2. Check last session file for details and context.
3. Continue from `Next Up` ticket.
